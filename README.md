# claude-env-ig-ingest

A personalized Claude Code environment system. Not a plugin — a framework for making Claude progressively more calibrated to how you work.

## What it is

Three files + one pipeline:

| File | Purpose |
|------|---------|
| `~/.claude/CLAUDE.md` | Behavior instructions injected into every Claude session |
| `~/.claude/lessons.md` | Auto-improving log — every error becomes a future prevention |
| `~/.claude/memory.md` | Your profile and preferences — Claude knows who it's talking to |
| `pipeline/transcribe.py` | Ingests video/image content (Instagram, YouTube, etc.) into `.md` notes |

## Core idea

Claude's default behavior is generic. This system makes it specific to you:
- It knows your stack, so it doesn't suggest alternatives you'd never use
- It knows your style, so it doesn't pad responses or add unrequested improvements
- It learns from mistakes, so errors don't repeat across sessions
- It manages its own context, so long sessions don't degrade in quality

## Setup

```bash
git clone https://github.com/YOUR_USERNAME/claude-env
cd claude-env
chmod +x setup.sh
./setup.sh
```

The setup script asks ~10 questions and generates your personal `~/.claude/` files. Takes 2 minutes.

**Dependencies:**
```bash
pip install -r pipeline/requirements.txt
sudo apt-get install ffmpeg        # Linux
brew install ffmpeg                # macOS
```

Also install [instaloader](https://instaloader.github.io/) for Instagram image posts:
```bash
pip install instaloader
```

## Pipeline: content → notes

Turns video and image content into structured `.md` files ready for a knowledge base.

```bash
export GROQ_API_KEY='gsk_...'

# Single URL
python3 pipeline/transcribe.py --url "https://www.instagram.com/reel/..." --topic "trading"

# Batch from file
python3 pipeline/transcribe.py --urls urls.txt --output ./notes --topic "psychology"
```

Supports:
- **Videos/Reels** → Groq Whisper transcription
- **Image posts** → Groq Llama 4 Vision extraction
- **Carousels** → per-image Vision extraction

Output is a `.md` file per post with frontmatter, transcription/content, and sections for personal notes and key ideas.

## Architecture

```
Two-layer design:
┌─────────────────────────────────────────┐
│  CORE (this repo — generic, reusable)   │
│  • Behavior principles                  │
│  • KERNEL prompt philosophy             │
│  • Subagent patterns                    │
│  • Pipeline tool                        │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│  IDENTITY (generated by setup.sh)       │
│  • Your stack defaults                  │
│  • Your active projects                 │
│  • Your collaboration preferences       │
│  • Your knowledge domains               │
└─────────────────────────────────────────┘
```

The core is reusable. The identity layer is what makes it yours.

## Key behaviors

**Plan before execute** — 3+ step tasks get a plan + confirmation before implementation.

**Lessons loop** — after any correction, Claude writes to `lessons.md`. Future sessions start with those patterns already in context.

**Reviewer agent** — code/system reviews use a zero-context agent to avoid confirmation bias.

**Context budget** — `/compact` is always called with explicit instructions to preserve architectural decisions, not just recent history.

**Bug autonomy** — bugs get fixed without asking for more context. Planning mode is for features, not for debugging.

## Customization

Edit `~/.claude/CLAUDE.md` directly at any time. The templates in `core/` are the starting point — your installed version is yours to evolve.

To regenerate from scratch: `./setup.sh` (will ask before overwriting).

## Trace2Skill — skills that learn from runs

Instead of writing skills by hand, run your agent 20 times, give one-sentence feedback per run, and let 4 parallel analyst agents extract the rules from the actual traces.

Based on the [Trace2Skill paper](https://arxiv.org/pdf/2603.25158) (Alibaba Qwen team). Skills built this way consistently outperform hand-written ones.

```
core/agents/          ← 4 analyst agents (error, success, structure, edge)
core/workflows/trace2skill.md  ← step-by-step workflow
```

See `core/workflows/trace2skill.md` for the full process.
